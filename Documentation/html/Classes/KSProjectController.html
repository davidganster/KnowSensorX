Inherits from NSObject

KSProjectController.m

Overview

Singleton class that manages all server interactions regarding projects, e.g. getting/creating projects and activities. Will listen for a kKSNotificationKeyServerReachable notification before becoming active. The KSProjectController periodically asks the server for new projects and updates all observers when the list changes.

updateProjectListWithDelay:

Internal method. Sends a request for all projects and all activities to the server and waits for a positive reply from the server. Upon completion of both calls, the project list and activity list will be updated, notifying all observers in the process. Regardless of the API calls worked, the KSProjectController will continue calling this method after timeIntervalBetweenPolls has passed. To stop polling, call stopUpdatingProjectList - this will set continuePolling to NO.

- (void)updateProjectListWithDelay:(double)delayInSeconds

Parameters

delayInSeconds

Amount of time in seconds to wait before starting the new poll.
