URLMappings

This class uses the URLMappings dictionary from the KSUserInfo class, but displays URLs in a sorted manner. To avoid sorting all the time, the URLs are sorted once and stored in this array. They are then used as keys for the URLMappings dictionary from KSUserInfo.

addMappingButtonPressed:

Creates the temporary KSURLMapping object (if none existed) and reload the tableView. This will cause the tableView to display the last row empty and in edit-mode. If the temporary mapping had already existed, this will simply put the temporary row in edit mode.

- (IBAction)addMappingButtonPressed:(id)sender

control:textView:doCommandBySelector:

This method has been overwritten to correctly handle ‘Tab’ key presses. The new behaviour always selects the next column, or switches to the next row/first column if possible. Backtabs are also handled here in a similar fashion, but backwards.

- (BOOL)control:(NSControl *)control textView:(NSTextView *)textView doCommandBySelector:(SEL)commandSelector

Parameters

control

The control that generated the event.

textView

The textView that registered the special key.

commandSelector

The selector that was caught by the control. (We only handle ‘insertTab:’ and ‘insertBacktab:’).

Return Value

YES iff the event was handled by us (namely when it’s a insertTab: or insertBacktab: event).

doubleClick

Sent when the user doubleclicks inside the tableView. If the click is detected to be in a valid row, this row (and column) is put in edit mode. Should the click have been below the last valid row, a new row will be added and edited.

- (void)doubleClick

removeMappingButtonPressed:

Removes the currently selected rows in the table view from the UI and deletes the corresponding values from KSUserInfo as well. The temporary mapping will be nil’d, should it be contained in the selection.
